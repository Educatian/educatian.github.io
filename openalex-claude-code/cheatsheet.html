04페이지네이션

  1. ≤ 200건: per-page=200 한 번이면 끝
  2. 200–10K: cursor=*로 시작, 응답 meta.next_cursor 다음 호출
  3. > 10K: page=N은 10K cap; cursor만 사용
  4. 전체 dump: 매월 ~330GB · S3 openalex-snapshot
안전 마진: time.sleep(0.05) (100/sec 하드 리밋)

06아이디어 → API 5단계

  1. RQ 한 문장으로
  2. 데이터 shape 테이블 형태로 그리기
  3. API 시퀀스: 어떤 entity → 어떤 filter
  4. 분석: aggregate / model / test
  5. 시각화 + 검증: robustness
한 프롬프트 = 한 단계. 5단계 한 번에 시키지 말 것.

07안 하는 것 Anti

  • 너무 넓은 첫 호출 ("education" → 50만 건)
  • 이름만으로 저자 매칭 (ORCID 없으면 동명이인)
  • WebFetch 한 번으로 1만 건 (페이지네이션 무시)
  • 인용 그래프 무한 hop (1-hop 제한)
  • Raw JSON 전체를 LLM에 (필요 필드만 select)
  • 인용수 절대값 단독 보고 (snapshot 날짜 명시)

08검수 체크리스트

  • 총 count 합리적인가 (0/100만 둘 다 의심)
  • 각 단계 결과를 file로 저장 (다음 단계 재사용)
  • None / 빈 배열 비율 보고
  • OPENALEX_KEY env var 설정됐나 (api_key 자동 주입)
  • "OpenAlex API, accessed YYYY-MM-DD" 명시
  • vault에는 정제 산출물만, 원시는 프로젝트

09자주 쓰는 프롬프트

  • 저자: ORCID 0000-...의 모든 works → JSONL
  • 인용: DOI ...를 인용한 2024+ 논문 표
  • 저널: ISSN로 source_id → 2015–25 paginate
  • 토픽: T14025 group_by=year
  • 알림: search "topic" + 지난 30일
